Updates Tags
Each league page must have a json data file with the following information as well as a HTML page containing only {{> league}} in the saved document, nothing else. In addition to the following attributes the page data file must also include a "title": "" and "page-category": {} information like all other pages. Lague banners must be located at assets/img/tournament/banners/ (.jpg if using default header, .png for custom banners), and map images in shared/maps/.png.
Main league attributes
Attribute Description Value Default
"name": The name of the tournament. String
"id": The ID used to reference the tournament assets. String
"heading": Display a text heading in the tournament banner. Tournaments without custom banners need this to be true. Boolean false
"host": The network or individual who hosted the tournament. String
"anounncement": The page which contains the tournament details. URL
"challonge": URL for the Challonge bracket. URL
"method": Tournament style. String
"selection": The amount of teams selected for the tournament and the method of selection (if appropriate). String
"players": The range of players per team roster permitted. String
"stages": [] The dates in which each stage took place. Stage info
"panels": [] Panels which can contain extra information or details about the tournament. This is not required. Optional information
"pool": [] The maps which were played during the tournament. Map info
"fisrt_place": The name of the team that finished first. Team name
"second_place": The name of the team that finished second. Team name
"third_place": The name of the team that finished third. Team name
"groups": [] The teams which participated in the tournament. Team info
Stage Information
Provides information about the different stages of the tournament.
Attribute Description Value Default
"stage": The name of the stage being played. String
"date": The starting date for this specific stage. String
Panels
Panels can contain pretty much anything regarding the tournament information that hasn't already been covered. Panels can come in two sizes; full which spans the whole screen width, or half which only covers half the screen width.
Attribute Description Value Default
"label": The label which is displayed as the panel title. String
"size": The size of the panel. Can either be full or half. Option string half
"rows": [] Contains the specifc rows which will be displayed on the panel
Rows sub-attributes
"label": The row label displayed on the left. This is optional and if left blank the content will span the entire row. String
"content": The text displayed inside of the row String
Map Pool Information
Provides information about the different maps that were played during the tournament. Image thumbnails for these maps must be put into a folder called pool inside another folder which is the same as the tournament ID. These images must have the same name as the slug reference ID of the map.
Attribute Description Value Default
"name": The name of the map. String
"slug": The unique ID for referencing this map in other places of the website, including the map image. String
"gamemode": The map's gamemode. String
"stage": The round in which this map was played. If the map was played in more than one round, create another map for that round. String
Group Information
Provides information about the teams that participated in the tournament and the group stages.
Attribute Description Value Default
"group": The name of the group which the teams have been put into - eg. Group A. String
"teams": [] The teams which are included in this group.
Teams sub-attributes
"team": The name of the team. String
Updating the tournament listing
Data for the tournament listing pages are seperate from the tournament's individual data file and is located within the (network)/index.json file.
Attribute Description Value Default
"name": The name of the tournament. String
"id": The ID used to reference the tournament assets and page. String
"heading": Display a text heading in the tournament banner. Tournaments without custom banners need this to be true. Boolean false
"disabled": Hide the tournament from the public listing. Boolean false
