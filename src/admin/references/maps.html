Data

Controls the maps being shown and how they are displayed to the browser. When adding new maps ensure that they are placed into the correct GitHub repository and that the folder name matches the slug entered into the data file. This reference is for map pages located in the src/maps/... directory of the website.

Please note that the last property of any root, object, or array property must not be followed by a comma. This will cause our JSON validator to throw red flags and will cause the automatic builds to fail.

Name Description Value Default
"data": {} Holds different sets of data to control the maps pages Object
Data Sub-properties
Name Description Value Default
"new": [] An array of new maps, giving each a blue glow and badge - references the map's slug Array
"popular": [] An array of popular maps, giving each a red glow and badge - references the map's slug Array
"featured": {} Defines settings for the featured maps panel Object
"maps": [] Defines each of the maps to display Array
Example
{
    "data": {
        "new": [
            "new_map_slug",
            "spaceship_battles"
        ],
        "popular": [
            "a_popular_map"
        ],
        "featured": { ... },
        "maps": [ ... ]
    }
}

Featured

Creates a green glowing panel above the search selection panel with a rotating carousel with the defined maps. Generally used to increase exposure to the selected maps during events such as a tournament or seasonal celebration. The amount of maps included in each panel should be limited to 12.

Featured sub-properties
Name Description Value Default
"enabled": Enables the featured panel Boolean "false"
"title": Custom title for the featured collection of maps String Featured Maps
"description": Custom description to further describe the collection of featured maps String
"maps": [] An array of maps to display in the featured panel - references the map's slug Array
Example
{
    "data": {
        "featured": {
            "enabled": true,
            "title": "Nostalgic Collection",
            "description": "A collection of nostalgic maps",
            "maps": [
                "race_for_victory",
                "total_war",
                "dynamo"
            ]
        }
    }
}

Map Objects

The individual maps being displayed on the page. When adding new maps ensure that they are placed into the correct GitHub repository and that the folder name matches the slug entered into the data file.

Name Description Value Default
"name": The name of the map String
"slug": The unique ID of the map which typically is a lowercase version of the map name with underscores in place of spaces and other punctuation String
"authors": [] The main authors of the map as listed in the XML document Array
"tags": [] Keywords to describe the map gamemode and play style Array
"locked": Disable all buttons and links for the map Boolean false
"discussion": The URL to the discussion/release thread or post URL
"contest": The contest ID to link to the record page if the map was part of a map making contest String
"sourced": If the map has a public download published by a documented author but hasn't provided ResourcePile with explicit permission to host the map
Note: If the map was published with a license, ignore this		 Boolean false
"commercial": If the map can be used for commercialised purposes Boolean
"download": If the map has a published download in the appropriate repository. Alternatively, if the main download is not hosted in our repositories, the URL to the download can be defined here as a URL string. Boolean or URL false
"xml": If the map has a published XML document in the appropriate repository Boolean false
"environment": The plugins that the map, more specifically the XML, is designed to be used for. Environments are arbitrary and are defined in the global data file. Environment Key Default Environment
"cardinal": If the map has been tested and works with the Cardinal plugin
Note: While this is still used, it has become obselete after the open-sourcing of the Project Ares plugins		 Boolean false
Examples
{
    "name": "Hydrolock II",
    "slug": "hydrolock_ii",
    "authors": [ ... ],
    "tags": [ ... ],
    "commercial": true,
    "download": true,
    "xml": true
}

Authors Objects

Name Description Value Default
"username": Most current username of the author String
"uuid": The UUID of the author UUID
Examples
{
    "authors": [
        {"username": "Lyzak", "uuid": "5986da63-a546-49c5-812d-d5c41a42510a"},
        {"username": "Plastix", "uuid": "ef4ea031-998f-4ec9-b7b6-1bdd428bcef8"},
        {"username": "StealthMedia", "uuid": "2a289d2a-d970-49c5-9a6c-01fc0264e317"}
    ]
}

Settings Maps Sub-properties

Maps Sub-properties
Name Description Value Default
"group": The group of maps, or network, that the page belongs to
This must match the first part of the maps repository, for example pgm for pgm-maps		 String
"default_environment": The plugins that the maps in the collection are designed to be used for. This will be the default environment and can be overriden per-map with "environment". Environments are arbitrary and are defined in the global data file. Environment Key
Example
{
    "settings": {
        "maps": {
            "group": "overcast",
            "default_environment": "pgm"
        }
    }
}
